# PC Monitor 使用说明

## 一、简介

PC Monitor 是一款轻量级实时系统性能监控工具，采用"采集端 + 显示端"分离架构。

- **采集端（collector.exe）**：运行在需要被监控的电脑上，采集 CPU、内存、GPU 数据
- **显示端（display.exe）**：运行在同一台或局域网内另一台电脑上，连接采集端实时显示数据
- **安卓端（PC-Monitor-Android.apk）**：安装在安卓手机上，连接采集端实时显示数据

采集端通过 TCP 协议（端口 9876）向外发送 JSON 数据，显示端和安卓端通过 IP 地址连接采集端获取数据。

## 三、采集端使用方法（collector.exe）

### 3.1 启动

直接双击 `collector.exe` 即可启动。

启动后会显示一个黑色命令行窗口，不要关闭它。窗口中会输出：
- 监听端口信息（默认 9876）
- 系统检测到的 CPU、内存、GPU 信息
- 连接的客户端数量

### 3.2 采集的数据

| 数据 | 来源 | 说明 |
|------|------|------|
| CPU 使用率 | sysinfo | 整体 CPU 占用百分比 |
| CPU 频率 | WMI | 实时主频（MHz），与任务管理器一致 |
| CPU 温度 | WMI | 热区温度（°C） |
| 内存使用率 | sysinfo | 已用/总量（MB） |
| GPU 使用率 | NVML | NVIDIA 显卡占用百分比 |
| GPU 温度 | NVML | 显卡温度（°C） |
| GPU 显存 | NVML | 已用/总量（MB） |
| GPU 功耗 | NVML | 实时功耗（W） |
| GPU 风扇 | NVML | 风扇转速百分比 |
| GPU 时钟 | NVML | 核心频率（MHz） |

> 注意：GPU 数据需要 NVIDIA 显卡及驱动。如果没有 NVIDIA 显卡，GPU 相关数据不会显示。

### 3.3 防火墙

首次运行时，Windows 防火墙可能会弹出提示，请点击"允许访问"。如果未弹出提示但显示端无法连接，请手动放行 9876 端口：

1. 打开"Windows Defender 防火墙" → "高级设置"
2. "入站规则" → "新建规则"
3. 选择"端口" → TCP → 特定端口填 `9876`
4. 允许连接 → 完成

## 四、Windows 显示端使用方法（display.exe）

### 4.1 启动

双击 `display.exe` 启动。

### 4.2 连接采集端

1. 启动后会显示连接界面
2. 在输入框中输入采集端所在电脑的 IP 地址（例如 `192.168.1.100`）
3. 点击 **Connect** 按钮
4. 连接成功后自动显示实时监控数据
5. IP 地址会自动保存，下次启动时自动连接

### 4.3 查看采集端电脑的 IP 地址

在采集端电脑上：
1. 按 `Win + R`，输入 `cmd` 回车
2. 输入 `ipconfig` 回车
3. 找到"无线局域网适配器 WLAN"或"以太网适配器"下的 `IPv4 地址`

### 4.4 界面说明

- **顶部栏**：显示连接地址，左侧圆形按钮切换主题，右侧叉号断开连接
- **仪表盘区域**：4 个弧形仪表盘分别显示 CPU、内存、GPU、显存占用百分比
- **详情面板**：CPU 频率/温度、GPU 温度/功耗/风扇转速的进度条
- **历史图表**：CPU、内存、GPU 使用率的实时折线图

### 4.5 主题切换

点击顶部栏左侧的圆形按钮，可在 4 种主题之间切换：
- **Dark**：深色主题（默认）
- **Tech Blue**：科技蓝
- **Cyberpunk**：赛博朋克
- **Minimal Light**：极简白

## 五、安卓端使用方法

### 5.1 安装

1. 将 `PC-Monitor-Android.apk` 传到手机（USB、微信、QQ 等均可）
2. 在手机上打开 APK 文件进行安装
3. 如果提示"不允许安装未知来源应用"，请在设置中开启允许

### 5.2 使用

1. 打开 PC Monitor 应用
2. 输入采集端电脑的 IP 地址
3. 点击 **Connect** 连接
4. IP 地址会自动保存，下次打开自动连接

### 5.3 界面说明

安卓端界面与 Windows 显示端保持一致：
- 弧形仪表盘显示 CPU、内存、GPU、显存占用
- 进度条显示频率、温度、功耗等详情
- 历史折线图显示趋势
- 支持 4 种主题切换

## 六、使用场景示例

### 场景 1：同一台电脑监控

采集端和显示端都在同一台电脑上运行：
1. 双击启动 `collector.exe`
2. 双击启动 `display.exe`
3. 输入 `127.0.0.1` 或 `localhost`，点击 Connect

### 场景 2：局域网内远程监控

A 电脑运行采集端，B 电脑或手机运行显示端：
1. 在 A 电脑上双击启动 `collector.exe`
2. 查看 A 电脑的 IP 地址（例如 `192.168.1.100`）
3. 在 B 电脑上双击启动 `display.exe`，输入 `192.168.1.100`，点击 Connect
4. 或者在手机上打开 PC Monitor 应用，输入 `192.168.1.100`，点击 Connect

### 场景 3：同时多个显示端

一台采集端可以同时被多个显示端连接：
1. 在被监控电脑上启动 `collector.exe`
2. 多台电脑或手机同时连接同一个 IP 地址

## 七、常见问题

**Q：连接不上怎么办？**
- 确认采集端已启动且命令行窗口未关闭
- 确认输入的 IP 地址正确
- 确认两台设备在同一局域网（同一 WiFi 或同一网段）
- 检查防火墙是否放行了 9876 端口

**Q：GPU 数据不显示？**
- 需要 NVIDIA 显卡及最新驱动
- AMD/Intel 核显暂不支持 GPU 监控，但 CPU 和内存数据正常

**Q：CPU 温度显示 N/A？**
- 部分电脑的 WMI 温度传感器不可用，属正常现象

**Q：安卓端连不上？**
- 确认手机和电脑在同一 WiFi 网络下
- 电脑端需要关闭防火墙或放行 9876 端口
- 尝试用手机浏览器访问 `http://电脑IP:9876`，如果能访问说明网络通

## 八、技术信息

- 采集端：Rust + sysinfo + NVML，约 0.9% CPU，40 MB 内存
- 显示端：Rust + iced GUI，约 0.02% CPU，27 MB 内存
- 安卓端：Flutter + Dart，支持 Android 6.0+
- 通信协议：TCP 端口 9876，JSON 格式，每秒推送一次数据
- 项目地址：https://github.com/hemo528/PC-monitor
